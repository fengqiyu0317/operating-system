# 程序运行输出示例

## 06stack_trace 程序运行输出

### 完整输出

```
[kernel] Loading app_6
====================================
RISC-V 调用栈打印程序
====================================

本程序演示如何打印调用栈信息
将创建多层函数调用，然后打印完整的调用栈

进入 level_1
进入 level_2
进入 level_3
递归深度: 5
递归深度: 4
递归深度: 3
递归深度: 2
递归深度: 1
到达递归底部，开始打印调用栈...

=== 调用栈追踪 ===

帧指针: 0x80208740

[0] 栈帧地址: 0x80208740
    返回地址: 0x804000d0
    上一个帧: 0x802087c0

栈帧地址: 0x802087c0
    返回地址: 0x804013e4
    上一个帧: 0x0

总共 2 个栈帧

====================================
程序执行完成
====================================
[kernel] Application exited with code 0
```

### 输出说明

1. **程序启动**
   - 显示程序标题和说明
   - 准备创建多层函数调用

2. **函数调用链**
   ```
   main() → level_1() → level_2() → level_3() → recursive_function(5)
   ```

3. **递归过程**
   - 从深度 5 递减到 1
   - 每层都打印当前递归深度

4. **调用栈追踪**
   - 显示当前帧指针地址: `0x80208740`
   - 遍历每个栈帧，显示：
     - 栈帧编号
     - 栈帧地址
     - 返回地址 (函数返回后要执行的指令地址)
     - 上一个帧的指针
   - 统计总栈帧数: 2 个

5. **程序退出**
   - 正常退出，返回码为 0

### 地址分析

- **栈帧地址**: `0x80208740`, `0x802087c0`
  - 这些是用户态栈空间的地址
  - 地址递减，符合栈的增长方向（从高地址到低地址）

- **返回地址**: `0x804000d0`, `0x804013e4`
  - 这些是代码段的地址
  - 指向函数调用后的下一条指令

### 调用栈链表结构

```
Stack Frame 0 (最内层)
├── Address: 0x80208740
├── Return Address: 0x804000d0 (recursive_function)
└── Caller FP: 0x802087c0 → Stack Frame 1

Stack Frame 1 (中间层)
├── Address: 0x802087c0
├── Return Address: 0x804013e4 (可能是 level_3 或其他函数)
└── Caller FP: 0x0 (链表结束)
```

### 实际应用价值

这个输出可以帮助开发者：

1. **调试崩溃**: 快速定位崩溃发生的函数调用链
2. **性能分析**: 了解函数调用的深度和路径
3. **内存分析**: 观察栈的使用情况
4. **程序理解**: 追踪复杂的函数调用关系

### 注意事项

- **终端编码**: 某些终端可能显示中文乱码，但不影响功能
- **地址有效性**: 所有返回地址都应指向代码段的合法指令
- **栈深度**: 实际应用中可能有更多栈帧，这里设置了深度限制

## 如何重现

```bash
# 1. 进入项目目录
cd chapter2-exercises

# 2. 编译用户程序
cd user && make build && cd ..

# 3. 编译并运行内核
cd os && make run

# 4. 观察输出，程序会自动运行所有应用
# 06stack_trace 是最后一个应用（app_6）
```
