# 第一章第四节：内核第一条指令（实践篇）操作指南

> 参考文档：https://rcore-os.cn/rCore-Tutorial-Book-v3/chapter1/4first-instruction-in-kernel2.html

## 本节目的

在 Qemu 上执行内核的第一条指令，验证内核镜像能否正确对接到 Qemu。

---

## 操作步骤概览

1. 编写内核第一条指令（汇编代码）
2. 在 main.rs 中嵌入汇编代码
3. 调整内核的内存布局（链接脚本）
4. 配置 Cargo 使用自定义链接脚本
5. 编译生成内核可执行文件
6. 生成内核镜像（剥离元数据）
7. 使用 GDB 验证启动流程

---

## 步骤 1：编写内核第一条指令

在 `os/src/` 目录下创建文件 `entry.asm`：

```asm
# os/src/entry.asm
    .section .text.entry
    .globl _start
_start:
    li x1, 100
```

**说明：**
- `.section .text.entry`：将代码放入 `.text.entry` 段（确保放在最低地址）
- `.globl _start`：声明 `_start` 为全局符号
- `li x1, 100`：将立即数 100 加载到寄存器 x1（用于后续验证）

---

## 步骤 2：在 main.rs 中嵌入汇编代码

修改 `os/src/main.rs`：

```rust
// os/src/main.rs
#![no_main]
#![no_std]

mod lang_items;

use core::arch::global_asm;
global_asm!(include_str!("entry.asm"));
```

**说明：**
- `use core::arch::global_asm;`：引入 global_asm 宏
- `global_asm!(include_str!("entry.asm"));`：将 entry.asm 内容嵌入代码中

---

## 步骤 3：创建链接脚本

在 `os/src/` 目录下创建文件 `linker.ld`：

```ld
# os/src/linker.ld
OUTPUT_ARCH(riscv)
ENTRY(_start)
BASE_ADDRESS = 0x80200000;

SECTIONS
{
    . = BASE_ADDRESS;
    skernel = .;

    stext = .;
    .text : {
        *(.text.entry)
        *(.text .text.*)
    }

    . = ALIGN(4K);
    etext = .;
    srodata = .;
    .rodata : {
        *(.rodata .rodata.*)
        *(.srodata .srodata.*)
    }

    . = ALIGN(4K);
    erodata = .;
    sdata = .;
    .data : {
        *(.data .data.*)
        *(.sdata .sdata.*)
    }

    . = ALIGN(4K);
    edata = .;
    .bss : {
        *(.bss.stack)
        sbss = .;
        *(.bss .bss.*)
        *(.sbss .sbss.*)
    }

    . = ALIGN(4K);
    ebss = .;
    ekernel = .;

    /DISCARD/ : {
        *(.eh_frame)
    }
}
```

**说明：**
- `BASE_ADDRESS = 0x80200000`：内核加载的起始地址
- `ENTRY(_start)`：程序入口点为 `_start` 符号
- `*(.text.entry)`：确保 `.text.entry` 段放在 `.text` 段最开头

---

## 步骤 4：配置 Cargo 使用链接脚本

修改 `os/.cargo/config` 文件：

```toml
# os/.cargo/config
[build]
target = "riscv64gc-unknown-none-elf"

[target.riscv64gc-unknown-none-elf]
rustflags = [
    "-Clink-arg=-Tsrc/linker.ld", "-Cforce-frame-pointers=yes"
]
```

**说明：**
- `-Clink-arg=-Tsrc/linker.ld`：指定使用自定义链接脚本
- `-Cforce-frame-pointers=yes`：保留帧指针，便于调试

---

## 步骤 5：编译生成内核可执行文件

在 `os` 目录下执行：

```bash
cd os
cargo build --release
```

编译完成后，可执行文件位于：
```
target/riscv64gc-unknown-none-elf/release/os
```

查看文件属性：
```bash
file target/riscv64gc-unknown-none-elf/release/os
```

预期输出：
```
target/riscv64gc-unknown-none-elf/release/os: ELF 64-bit LSB executable, UCB RISC-V, version 1 (SYSV), statically linked, not stripped
```

---

## 步骤 6：生成内核镜像

由于 Qemu 的文件加载功能不支持完整的可执行文件格式，需要剥离元数据：

```bash
rust-objcopy --strip-all target/riscv64gc-unknown-none-elf/release/os -O binary target/riscv64gc-unknown-none-elf/release/os.bin
```

查看文件大小对比：
```bash
stat target/riscv64gc-unknown-none-elf/release/os
stat target/riscv64gc-unknown-none-elf/release/os.bin
```

预期结果：
- `os` 文件约 1016 字节（包含元数据）
- `os.bin` 文件约 4 字节（仅包含一条指令）

---

## 步骤 7：使用 GDB 验证启动流程

### 7.1 准备 bootloader

确保存在 `bootloader/rustsbi-qemu.bin` 文件。如果还没有，可以从 rCore 项目中获取。

### 7.2 启动 Qemu（调试模式）

在 `os` 目录下执行：

```bash
qemu-system-riscv64 \
    -machine virt \
    -nographic \
    -bios ../bootloader/rustsbi-qemu.bin \
    -device loader,file=target/riscv64gc-unknown-none-elf/release/os.bin,addr=0x80200000 \
    -s -S
```

**参数说明：**
- `-machine virt`：使用 Qemu 的 virt 机器
- `-nographic`：不使用图形界面
- `-bios`：指定 bootloader
- `-device loader,file=...,addr=...`：加载内核镜像到指定地址
- `-s`：监听 TCP 端口 1234 等待 GDB 连接
- `-S`：Qemu 启动后暂停，等待 GDB 命令

### 7.3 启动 GDB 客户端

**打开另一个终端**，在 `os` 目录下执行：

```bash
riscv64-unknown-elf-gdb \
    -ex 'file target/riscv64gc-unknown-none-elf/release/os' \
    -ex 'set arch riscv:rv64' \
    -ex 'target remote localhost:1234'
```

### 7.4 GDB 调试命令

进入 GDB 后，可以使用以下命令：

```gdb
# 查看 Qemu 固件内容
x/10i $pc

# 单步执行
si

# 查看寄存器 t0 的值（16进制）
p/x $t0

# 在内核入口处设置断点
b *0x80200000

# 继续执行到断点
c

# 查看内核入口处指令
x/5i $pc

# 单步执行内核第一条指令
si

# 验证 x1 寄存器的值（应为 100）
p/d $x1
```

### 7.5 验证结果

如果一切正常，你应该看到：

1. 在 `0x80200000` 处的指令是 `li ra,100`（即 `li x1, 100`）
2. 执行该指令后，`x1` 寄存器的值为 `100`

```
(gdb) x/5i $pc
=> 0x80200000:      li      ra,100
0x80200004: unimp
...

(gdb) si
0x0000000080200004 in ?? ()

(gdb) p/d $x1
$2 = 100
```

## 常见问题

### Q1: 找不到 `rust-objcopy` 命令

确保已安装 Rust 工具链，并添加到 PATH：
```bash
rustup component add llvm-tools
```

或使用系统的 `objcopy`：
```bash
riscv64-unknown-elf-objcopy --strip-all target/riscv64gc-unknown-none-elf/release/os -O binary target/riscv64gc-unknown-none-elf/release/os.bin
```

### Q2: 找不到 `riscv64-unknown-elf-gdb`

需要安装 RISC-V 交叉编译工具链：
```bash
# Ubuntu
sudo apt install gcc-riscv64-unknown-elf gdb-multiarch
```

或使用 `gdb-multiarch` 替代：
```bash
gdb-multiarch \
    -ex 'file target/riscv64gc-unknown-none-elf/release/os' \
    -ex 'set arch riscv:rv64' \
    -ex 'target remote localhost:1234'
```

### Q3: Qemu 直接运行（不调试）

如果想直接运行 Qemu 而不使用 GDB 调试，去掉 `-s -S` 参数：

```bash
qemu-system-riscv64 \
    -machine virt \
    -nographic \
    -bios ../bootloader/rustsbi-qemu.bin \
    -device loader,file=target/riscv64gc-unknown-none-elf/release/os.bin,addr=0x80200000
```

## 文件结构总结

完成本节后，`os` 目录结构如下：

```
os/
├── .cargo/
│   └── config          # Cargo 配置（含链接脚本路径）
├── Cargo.toml
├── src/
│   ├── entry.asm       # 内核第一条指令（汇编）
│   ├── lang_items.rs   # 语言项（panic 处理等）
│   ├── linker.ld       # 链接脚本
│   └── main.rs         # 内核主入口
└── target/
    └── riscv64gc-unknown-none-elf/
        └── release/
            ├── os          # 内核可执行文件（ELF格式）
            └── os.bin      # 内核镜像（二进制）
```
