## 🛠️ 动手操作步骤

### 步骤 1：创建一个简单的 Rust 项目

```bash
# 创建一个新的 Rust 可执行程序项目
cargo new os --bin

# 查看项目结构
tree os
```

**预期输出：**
```
os
├── Cargo.toml
└── src
    └── main.rs

1 directory, 2 files
```

---

### 步骤 2：查看并运行默认程序

```bash
# 进入项目目录
cd os

# 查看默认的 main.rs 内容
cat src/main.rs
```

**默认内容：**
```rust
fn main() {
    println!("Hello, world!");
}
```

```bash
# 编译并运行
cargo run
```

**预期输出：**
```
   Compiling os v0.1.0 (...)
    Finished dev [unoptimized + debuginfo] target(s) in 1.15s
     Running `target/debug/os`
Hello, world!
```

---

### 步骤 3：观察系统调用（理解执行环境）

这一步帮助你理解 `Hello, world!` 背后发生了什么。

```bash
# 使用 strace 工具跟踪系统调用
strace target/debug/os
```

**观察要点：**
- 找到 `write(1, "Hello, world!\n", 14)` 系统调用（输出字符串）
- 找到 `exit_group(0)` 系统调用（程序退出）
- 注意：程序运行前后有大量其他系统调用用于初始化

**扩展实验：创建一个"什么都不做"的程序**

```bash
# 创建一个空程序
cd ..
cat > donothing.rs << 'EOF'
fn main() {
    // do nothing
}
EOF

# 编译
rustc donothing.rs

# 运行（无输出）
./donothing

# 用 strace 观察
strace ./donothing
```

**思考：** 为什么一个空程序也需要几十次系统调用？

---

### 步骤 4：查看当前 Rust 编译器的目标平台

```bash
# 查看 rustc 版本和默认目标平台
rustc --version --verbose
```

**关注输出中的 `host` 字段：**
```
host: x86_64-unknown-linux-gnu
```

**目标三元组解读：**
| 部分 | 含义 |
|------|------|
| `x86_64` | CPU 架构 |
| `unknown` | CPU 厂商 |
| `linux` | 操作系统 |
| `gnu` | 运行时库（GNU libc） |

---

### 步骤 5：查看 Rust 支持的 RISC-V 目标平台

```bash
# 列出所有支持的 RISC-V 目标
rustc --print target-list | grep riscv
```

**预期输出：**
```
riscv32gc-unknown-linux-gnu
riscv32i-unknown-none-elf
riscv32imac-unknown-none-elf
riscv32imc-unknown-none-elf
riscv64gc-unknown-linux-gnu
riscv64gc-unknown-none-elf    # ← 我们将使用这个
riscv64imac-unknown-none-elf
```

**目标平台解读：**
| 目标 | 含义 |
|------|------|
| `riscv64gc` | RISC-V 64位，支持 IMAFD 拓展 |
| `unknown` | 厂商未知 |
| `none` | 无操作系统（裸机） |
| `elf` | 生成 ELF 格式，无标准运行时库 |

---

### 步骤 6：尝试交叉编译到 RISC-V（重要！）

这一步将揭示裸机编程的第一个挑战。

```bash
# 回到 os 项目目录
cd os

# 尝试为 RISC-V 裸机平台编译
cargo run --target riscv64gc-unknown-none-elf
```

**预期错误：**
```
error[E0463]: can't find crate for `std`
  |
  = note: the `riscv64gc-unknown-none-elf` target may not be installed
```

**错误原因：**
- `riscv64gc-unknown-none-elf` 是裸机平台，没有操作系统支持
- Rust 标准库 `std` 依赖操作系统提供的系统调用
- 在裸机平台上无法使用标准库

---

### 步骤 7：安装 RISC-V 目标支持（如果尚未安装）

```bash
# 安装 RISC-V 目标
rustup target add riscv64gc-unknown-none-elf

# 再次尝试编译（仍然会报错，但原因不同）
cargo build --target riscv64gc-unknown-none-elf
```

## 📝 本节关键概念总结

### 执行环境栈（从上到下）

```
┌─────────────────────────────────┐
│        应用程序 (Application)     │  ← 我们写的代码
├─────────────────────────────────┤
│     函数库 (Standard Library)    │  ← println! 等功能
├─────────────────────────────────┤
│      操作系统内核 (Kernel)        │  ← 系统调用
├─────────────────────────────────┤
│        硬件 (Hardware)           │  ← CPU/内存/I/O
└─────────────────────────────────┘
```

### RISC-V 指令集拓展

| 拓展 | 说明 |
|------|------|
| I | 基本整数指令集（必须） |
| M | 整数乘除法 |
| A | 原子操作 |
| F | 单精度浮点 |
| D | 双精度浮点 |
| C | 压缩指令 |
| G | IMAFD 的组合 |

**`riscv64gc` = `riscv64imafdc`**

### std vs core

| 特性 | std（标准库） | core（核心库） |
|------|--------------|---------------|
| 需要操作系统 | ✅ 是 | ❌ 否 |
| 可用于裸机 | ❌ 否 | ✅ 是 |
| 功能丰富度 | 完整 | 精简 |
| 适用场景 | 应用开发 | 内核/嵌入式 |
