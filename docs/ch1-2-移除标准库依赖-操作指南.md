# 第一章第2节：移除标准库依赖 - 操作指南

## 📖 本节目标

构建一个内核最小执行环境，使其能在 RV64GC（RISC-V 64位CPU）裸机上运行。本节的核心任务是**移除 Rust 标准库 std 的依赖**，转而使用核心库 core（不需要操作系统支持）。

---

## 🔧 操作步骤

### 步骤 1：添加 RISC-V 64 目标平台

首先需要给 `rustc` 编译器添加目标平台支持：

```bash
rustup target add riscv64gc-unknown-none-elf
```

### 步骤 2：创建 Cargo 配置文件

在 `os` 目录下新建 `.cargo` 目录，并在其中创建 `config` 文件：

```bash
mkdir -p os/.cargo
```

创建文件 `os/.cargo/config`，内容如下：

```toml
# os/.cargo/config
[build]
target = "riscv64gc-unknown-none-elf"
```

> **说明**：这会使 Cargo 在 os 目录下默认使用 `riscv64gc` 作为目标平台，而不是默认的 `x86_64-unknown-linux-gnu`。这是一种**交叉编译**（Cross Compile）的场景。

### 步骤 3：修改 main.rs - 禁用标准库

在 `os/src/main.rs` 文件的开头添加：

```rust
#![no_std]
```

这告诉 Rust 编译器不使用 Rust 标准库 std，转而使用核心库 core。

### 步骤 4：移除 println! 宏

由于 `println!` 宏由标准库 std 提供，需要系统调用支持，因此需要**注释掉或删除** `println!` 相关代码：

```rust
// 注释掉或删除这行
// println!("Hello, world!");
```

### 步骤 5：创建 panic 处理函数

创建新文件 `os/src/lang_items.rs`，实现 panic 处理函数：

```rust
// os/src/lang_items.rs
use core::panic::PanicInfo;

#[panic_handler]
fn panic(_info: &PanicInfo) -> ! {
    loop {}
}
```

然后在 `os/src/main.rs` 中添加模块声明：

```rust
// os/src/main.rs
#![no_std]
mod lang_items;
// ... other code
```

> **说明**：`#[panic_handler]` 是编译指导属性，用于标记核心库 core 中的 `panic!` 宏要对接的函数。该函数需要具有 `fn(&PanicInfo) -> !` 的函数签名。

### 步骤 6：移除 main 函数

在 `os/src/main.rs` 的开头添加：

```rust
#![no_main]
```

并**删除原来的 `main` 函数**。

> **说明**：`start` 语义项代表标准库 std 在执行应用程序之前需要的初始化工作。由于我们禁用了标准库，最简单的解决方案是使用 `#![no_main]` 告诉编译器我们没有一般意义上的 `main` 函数。

## 📚 概念说明

### 什么是 LibOS（库操作系统）？

LibOS 以函数库的形式存在，为应用程序提供操作系统的基本功能。它最早来源于 MIT PDOS 研究小组的 Exokernel（外核）操作系统结构研究。

### 本地编译 vs 交叉编译

| 类型 | 说明 |
|------|------|
| **本地编译** | 在当前开发平台编译，生成的程序也在该平台运行（如 Linux x86-64 → Linux x86-64） |
| **交叉编译** | 在一种平台上编译出在另一种平台上运行的程序（如 Linux x86-64 → rCore/riscv64gc） |

### std vs core

| 库 | 特点 |
|----|------|
| **std（标准库）** | 需要操作系统支持，提供 println!、Vec、String 等 |
| **core（核心库）** | 不需要操作系统支持，只提供最基本的类型和功能 |

## ⚠️ 常见问题

### Q: 编译报错 `cannot find macro 'println' in this scope`
**A**: 需要注释掉或删除代码中的 `println!` 宏调用。

### Q: 编译报错 `#[panic_handler] function required, but not found`
**A**: 需要创建 `lang_items.rs` 文件并实现 panic 处理函数，同时在 `main.rs` 中声明该模块。

### Q: 编译报错 `requires 'start' lang_item`
**A**: 需要在 `main.rs` 开头添加 `#![no_main]` 并删除原来的 `main` 函数。
